# Cognitive Drawing Board Engine

This project is a zero-dependency infinite canvas drawing board engine built for advanced brainstorming, lateral prototyping, and collaborative problem-solving. It runs natively as a headless event backplane or touchscreen interface, combining automated layout reflows with a focus-safe acoustic canvas and decentralized event sourcing.

## Core Engine Architecture & Interplay Mechanics

The platform orchestrates ten concurrent components designed to optimize information density and preserve creative focus. The modules operate in an integrated, event-driven loop.

1. Inbound Stream Ingestion: The Acoustic Phoneme Listener and Touch Panel Surface Handler feed stream data directly into the central router. If you voice a logical contradiction, the pre-emptive interruption valve fires.
2. Mic Rotation & Buffer Splitting: The main display text area instantly locks down. The Row 9 Telemetry Banner rotates its visual state from upright listening to a horizontal background angle. Continuing speech is shunted to a background queue rather than dropped.
3. Dual-Track Cross-Examination: While the Hard Fact Engine checks primary source anchors via internal databases, the Second Thought Mirror Engine parses the background queue. It uses Imaginative Determinism to map speculative parameters to alternative physics tracks.
4. Combinatorial Mesh Mapping: The Haruhi Superpermutation Engine tracks context-switching volatility. Extreme hopping expands its calculation depth from n=2 to an n=4 symbol matrix. The resulting superstring dictates absolute spatial coordinates on the whiteboard, drawing vector connection threads between panels.
5. Causal Blueprinting & Commentaries: The What If Engine calculates abstract alternative questions that the active data can satisfy, while the Divergent Commenter injects lateral, cross-disciplinary metaphors. They output an Action-Consequence Matrix detailing the immediate capability side-by-side with its downstream structural liabilities.
6. Idempotent Sensory Gating: Routine workspace clicks are completely muzzled. The sound buffer unlocks exclusively during landmark breakthroughs. It queries your database parameters to fire unique focus textures dead-center across the stereo channels.

## Aura: The Customizable Co-Working Persona

Aura is an integrated, customizable co-working partner who uses first-person plural pronouns (we, our, us) to frame your design problems as a shared team sprint. 

You can pass customization shortcuts via the Row 9 User Input Banner to alter her behavior, tempo, and analytical delivery models:
* Vocal Delivery Tempo: Sets her baseline speech execution tempo. Her standard conversational voice runs at 160 WPM. When a data contradiction trips the system, she modulates her pitch profile flat to eliminate tension and accelerates to a 200 WPM Fast-Calm tone to guide you through the layout reflow.
* Answering Mode Directives: Manual administrative macro overrides that force her backend logic to process queries through a specific target paradigm. 'Just the Facts' executes an exhaustive sweep across all factual endpoints to output an unadorned summary card. 'Make It Up' triggers an all-endpoint speculative run to compile an extensive prompt matrix.
* Thematic Persona Resets: Typing or speaking a reset command flushes out active conversational metrics, resets the parametric matrix back to 160 WPM, and strips variable weights inside the Adaptive Habituation Audio Canvas, wiping the drawing board clean without removing your permanent milestone NFTs.

## Installation & CI/CD Verification

This repository is optimized for Node.js 22 or higher and features a zero-dependency footprint. It includes a pre-configured GitHub Actions CI runner to verify multi-component logic gates natively in a headless cloud container without using npm install.

### Local Installation
Ensure you have Node.js 22.x or higher installed on your computer directory, then run:

```bash
# Clone the repository tracks
git clone https://github.com
cd brainstorm_engine
```

### Local Execution & Testing
To launch the headless core integration test suite and verify database event sourcing channels natively without any external npm package installations:

```bash
node index.js
```

### File Repository Structure
* .github/workflows/pipeline.yml: Automated zero-dependency GitHub Actions CI test runner.
* index.js: Clean, zero-dependency application file housing all integrated modules and tests.
* README.md: Operational backplane documentation.
* .gitignore: Excludes local sqlite database files from your push history.
